## workspaces

a workspace is a folder you're working in. switch workspaces with `⌘O` / `ctrl+O` or by clicking the sidebar header.

your home folder (`~/.spacecake`) is always available for notes that aren't tied to a specific project.

## files and tabs

- **quick open**: press `⌘P` / `ctrl+P` to search and open files by name
- **new file**: press `⌘N` / `ctrl+N` to create a new file
- **close tab**: press `⌘W` / `ctrl+W` or click the × on the tab
- **save**: press `⌘S` / `ctrl+S` to save the current file

tabs show open files at the top of the editor. scroll horizontally if you have many tabs open.

## autosave

spacecake can automatically save files as you edit. enable autosave in workspace settings:

1. click the gear icon in the sidebar (or navigate to Settings)
2. toggle **enable autosave** on

when enabled, changes are saved to disk automatically - no need to press `⌘S` / `ctrl+S`.

## panels

spacecake has three panels — terminal, git, and tasks. each panel can be docked to the left, bottom, or right of the editor. to change a panel's dock position, right-click its toggle button in the status bar and choose a new position.

## terminal

the integrated terminal is docked to the right of the editor by default. toggle it with `` ctrl+` ``, or click and drag the edge to resize. it supports multiple tabs — press `⌘T` / `ctrl+T` to open a new tab, and `ctrl+Tab` to switch between them.

see the [terminal guide](/terminal) for the full details on tabs, shell integration, and keyboard shortcuts.

## git

the git panel (docked to the left by default) gives you a visual interface for common git operations. it has two tabs:

- **changes** — shows modified, staged, untracked, and deleted files. use the checkboxes to select which files to include in a commit, type a message, and hit commit. you can also discard changes per-file or all at once.
- **history** — lists recent commits. select a commit to see the files it changed, and click a file to view its diff.

the toolbar includes fetch, pull, and push buttons, along with a branch switcher for checking out or creating branches. ahead/behind indicators show how your local branch compares to the remote.

clicking a changed file opens an inline diff in the editor.
